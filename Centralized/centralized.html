What is Centralized Algorithm?

A centralized algorithm is a type of routing algorithm that has access to complete information about the network's topology and link costs. It computes the least-cost path between course and destination with full knowledge of the network.


Centralized algorithms can be implemented either at a logically centralized controller, where calculations are completed at one site and the resulting paths are distributed to routers, or in the routing component of every router, where each router performs its own computations using the complete network information.


Link-state (LS) algorithms are a type of centralized routing algorithm that use complete knowledge of the network's topology and link costs to determine the shortest path between two nodes. Each router maintains a map of the network and shares this link cost information between routers. The algorithm computes the shotest path using this global state information.


How does dijkstra's algorithm work?

  2. There is a forwarding table for that source node
  3. Have two lists

  5. The other keeps track of the nodes that have not been visited
  6. The algorithm starts by visiting the node with the smallest known distance from the starting node, which would be the source node.
  7. Then the algorithm inspects the unvisited nodes neighboring the node it is currently at
  8. From the current node the distance between each neighbor is calculated
